Derivation and Implementation of a Dense Neural Network on MNIST

Created2022-10-31|Updated2022-11-05|Study Notes
|Word Count:4.9k|Reading Time:30mins|Post Views:

The main purpose of backpropagation is to calculate the partial derivatives of the neural-network error with respect to parameters such as biases and weights, allowing gradient descent to be performed. The first half of this article derives the algorithm; the second half uses a fully connected neural network and the MNIST dataset to implement handwritten-digit recognition.

This algorithm was still very difficult for me to understand, so I wrote these notes to prevent myself from forgetting it. Another reason is that neural-network formulas have so many superscripts and subscripts that it is very easy to write one incorrectly in a physical notebook. If you do not yet have a basic understanding of neural networks, I recommend 3Blue1Brown’s neural-network video series.

I must also say that MqCreaple is truly incredible: after watching the videos, he directly derived every formula by hand—even more astonishingly, he managed to teach someone like me.

Formula Derivation

Notation and Language Conventions

  • σ()\sigma()σ() denotes the activation function.
  • EEE denotes the final error.
  • err⁡()\operatorname{err}()err() denotes the error function.
  • y^\hat yy^​ denotes the neural network’s prediction, while yyy denotes the actual answer.
  • lll denotes a neural-network layer; a smaller value means the layer is closer to the input layer.
  • wji[l]w^{[l]}_{ji}wji[l]​ denotes an edge from node jjj in layer lll to node iii in layer l−1l-1l−1.
  • bi[l]b^{[l]}_ibi[l]​ denotes the bias of node iii in layer lll.
  • zi[l]z^{[l]}_izi[l]​ denotes the output of node iii in layer lll before applying the activation function.
  • ai[l]a^{[l]}_iai[l]​ denotes σ(zi[l])\sigma(z^{[l]}_i)σ(zi[l]​), the node’s output after applying the activation function.
  • nln^lnl denotes the number of nodes in layer lll.
  • An underline beneath a variable indicates that it is treated as a constant, such as x‾\underline{x}x​.
  • A layer “before” another layer has a smaller lll, and a layer “after” it has a larger lll.

A Neural Network with One Node per Layer

First consider the simplest fully connected neural network, in which every layer has only one node. The following diagram represents the process of calculating a single node’s output a[l]a^{[l]}a[l]. The variable at the starting point of an arrow and the corresponding function produce the variable at the arrow’s endpoint.

    graph TB
 alm1["a(l-1)"] & w & b --> z --> al["a(l)"] --> Error
 y-->Error

Written as functions, the process is:

z[l]=wa[l−1]+ba[l]=σ(z[l])E=err⁡(a[l],y)z^{[l]} = wa^{[l-1]} + b \\ a^{[l]} = \sigma(z^{[l]}) \\ E = \operatorname{err}(a^{[l]}, y) z[l]=wa[l−1]+ba[l]=σ(z[l])E=err(a[l],y)

To perform gradient descent on the weight w[l]w^{[l]}w[l] according to the error, we need the partial derivative of the error with respect to the weight:

∂E∂w[l]\frac{\partial E}{\partial w^{[l]}} ∂w[l]∂E​

We use a partial derivative because the calculation of z[l]z^{[l]}z[l] depends on three variables, while we want to know how changing w[l]w^{[l]}w[l] affects the error.

When taking the partial derivative, assume that the other variables are constants. Only one variable changes, along with the variables directly affected by it—in this case, the chain w[l]→z[l]→a[l]→Ew^{[l]}\to z^{[l]}\to a^{[l]}\to Ew[l]→z[l]→a[l]→E. Constants are underlined in the following expression:

E=err⁡(σ(w[l]a[l−1]‾+b‾),y‾)E = \operatorname{err}(\sigma(w^{[l]}\underline{a^{[l-1]}} + \underline{b}), \underline{y}) E=err(σ(w[l]a[l−1]​+b​),y​)

We can now apply the chain rule:

E′=err⁡′(σ(w[l]a[l−1]‾+b‾),y‾)⋅σ′(w[l]a[l−1]‾+b‾)⋅(w[l]a[l−1]‾+b‾)′\begin{align*} E^\prime = \operatorname{err}^\prime(&\sigma(w^{[l]}\underline{a^{[l-1]}} + \underline{b}), \underline{y}) \\ \cdot &\sigma^{\prime}(w^{[l]}\underline{a^{[l-1]}} + \underline{b}) \\ \cdot &(w^{[l]}\underline{a^{[l-1]}} + \underline{b})^\prime \end{align*} E′=err′(⋅⋅​σ(w[l]a[l−1]​+b​),y​)σ′(w[l]a[l−1]​+b​)(w[l]a[l−1]​+b​)′​

In another form, which will be more convenient later:

∂E∂w[l]=∂z[l]∂w[l]⋅∂a[l]∂z[l]⋅∂E∂a[l]\frac{\partial E}{\partial w^{[l]}} = \frac{\partial z^{[l]}}{\partial w^{[l]}} \cdot \frac{\partial a^{[l]}}{\partial z^{[l]}} \cdot \frac{\partial E}{\partial a^{[l]}} ∂w[l]∂E​=∂w[l]∂z[l]​⋅∂z[l]∂a[l]​⋅∂a[l]∂E​

The intermediate partial derivatives in the chain rule can then be calculated. Assuming the error function is squared error, the expression becomes:

∂z[l]∂w[l]=(w[l]a[l−1]‾+b‾)′=a[l−1]∂a[l]∂z[l]=σ′(z[l])∂E∂a[l]=2(a[l]−y)\begin{align*} \frac{\partial z^{[l]}}{\partial w^{[l]}} &= (w^{[l]}\underline{a^{[l-1]}} + \underline{b})^\prime = a^{[l-1]} \\ \frac{\partial a^{[l]}}{\partial z^{[l]}} &= \sigma^{\prime}(z^{[l]}) \\ \frac{\partial E}{\partial a^{[l]}} &= 2(a^{[l]} - y) \end{align*} ∂w[l]∂z[l]​∂z[l]∂a[l]​∂a[l]∂E​​=(w[l]a[l−1]​+b​)′=a[l−1]=σ′(z[l])=2(a[l]−y)​

Be careful not to reverse 2(a[l]−y)2(a^{[l]}-y)2(a[l]−y). For example, when a[l]a^{[l]}a[l] is too large, we also want the derivative to be large so that the value being adjusted can have the derivative subtracted from it.

The preceding calculation gives the error’s partial derivative with respect to the weight. For the bias and the previous layer’s output, simply replace ∂z[l]∂w[l]\frac{\partial z^{[l]}}{\partial w^{[l]}}∂w[l]∂z[l]​ in the formula. In other words, let the previous layer’s output or this layer’s bias affect z[l]z^{[l]}z[l] instead of the weight.

For b[l]b^{[l]}b[l], use:

∂z[l]∂b[l]=(w[l]a[l−1]‾+b)′=1\frac{\partial z^{[l]}}{\partial b^{[l]}} = (\underline{w^{[l]}a^{[l-1]}} + b)^\prime = 1 ∂b[l]∂z[l]​=(w[l]a[l−1]​+b)′=1

For a[l−1]a^{[l-1]}a[l−1], use:

∂z[l]∂a[l−1]=(w[l]‾a[l−1]+b‾)′=w[l]\frac{\partial z^{[l]}}{\partial a^{[l-1]}} = (\underline{w^{[l]}}a^{[l-1]} + \underline{b})^\prime = w^{[l]} ∂a[l−1]∂z[l]​=(w[l]​a[l−1]+b​)′=w[l]

Now consider the following network:

    graph TB
 al["a(l)"] & wlp["w(l + 1)"] & blp["b(l + 1)"] --> zlp["z(l + 1)"] --> alp["a(l + 1)"] --> ...many other layers
 __["w(l + 2)"] & _["b(l + 2)"]-->...many other layers

Here, the layer after a[l]a^{[l]}a[l] does not connect directly to the error function; many layers intervene. We can no longer calculate ∂E∂a[l]\frac{\partial E}{\partial a^{[l]}}∂a[l]∂E​ directly, and therefore cannot directly calculate the partial derivatives for w[l]w^{[l]}w[l] and b[l]b^{[l]}b[l], because EEE lies many layers later. This is where backpropagation is required.

We know:

∂E∂a[l]=∂z[l+1]∂a[l]⋅∂a[l+1]∂z[l+1]⋅∂E∂a[l+1]\frac{\partial E}{\partial a^{[l]}} = \frac{\partial z^{[l+1]}}{\partial a^{[l]}} \cdot \frac{\partial a^{[l+1]}}{\partial z^{[l+1]}} \cdot \frac{\partial E}{\partial a^{[l+1]}} ∂a[l]∂E​=∂a[l]∂z[l+1]​⋅∂z[l+1]∂a[l+1]​⋅∂a[l+1]∂E​

The expression shows that ∂E∂a[l]\frac{\partial E}{\partial a^{[l]}}∂a[l]∂E​ for an earlier layer can be derived from a later layer. Before calculating partial derivatives for weights and biases, start from the output layer and propagate ∂E∂a[l]\frac{\partial E}{\partial a^{[l]}}∂a[l]∂E​ backward one layer at a time.

Multiple Nodes per Layer

The backpropagation process in the preceding example is clear and involves no linear algebra. In a real neural network, however, every layer contains multiple nodes:

    graph LR
     l1["(l-1)1"] & l2["(l-1)2"] & l3["(l-1)3"] ---> lp1["l1"] & lp2["l2"] & lp3["l3"]

Partial Derivative of the Error with Respect to a Weight

wji[l]w^{[l]}_{ji}wji[l]​ denotes an edge connecting node jjj in layer lll to node iii in layer l−1l-1l−1. How do we calculate ∂E∂wji[l]\frac{\partial E}{\partial w^{[l]}_{ji}}∂wji[l]​∂E​?

We can still use the original formula, because a layer with multiple nodes is fundamentally composed of multiple single-node layers. We must, however, pay attention to the subscripts:

∂E∂wji[l]=∂zj[l]∂wji[l]⋅∂aj[l]∂zj[l]⋅∂E∂aj[l]=ai[l−1]⋅σ′(zj[l])⋅∂E∂aj[l]\begin{align*} \frac{\partial E}{\partial w^{[l]}_{ji}} &= \frac{\partial z^{[l]}_j}{\partial w^{[l]}_{ji}} \cdot \frac{\partial a^{[l]}_j}{\partial z^{[l]}_j} \cdot \frac{\partial E}{\partial a^{[l]}_j} \\ &= a^{[l-1]}_i \cdot \sigma^\prime(z^{[l]}_j) \cdot \frac{\partial E}{\partial a^{[l]}_j} \end{align*} ∂wji[l]​∂E​​=∂wji[l]​∂zj[l]​​⋅∂zj[l]​∂aj[l]​​⋅∂aj[l]​∂E​=ai[l−1]​⋅σ′(zj[l]​)⋅∂aj[l]​∂E​​

Every variable related to layer l−1l-1l−1 uses subscript iii, such as ai[l−1]a^{[l-1]}_iai[l−1]​. Intuitively, when a unit of weight changes, a larger input from the previous layer has a larger effect on the final error function. Variables related to layer lll use subscript jjj.

Because σ′(zj[l])∂E∂aj[l]\sigma^\prime(z^{[l]}_j)\frac{\partial E}{\partial a^{[l]}_j}σ′(zj[l]​)∂aj[l]​∂E​ uses the same subscript throughout, call it rjr_jrj​ to make the matrix-operation formula easier to write.

Rewriting the formula gives:

∂E∂wji[l]=rjai[l−1]\frac{\partial E}{\partial w^{[l]}_{ji}} = r_j a^{[l-1]}_i ∂wji[l]​∂E​=rj​ai[l−1]​

In the matrix form of w[l]w^{[l]}w[l], jjj increases by row and iii increases by column. The derivative matrix is:

∂E∂w[l]=[r1a1[l−1]r1a2[l−1]⋯r1anl−1[l−1]r2a1[l−1]r2a2[l−1]⋯r2anl−1[l−1]⋮⋮⋱⋮rnla1[l−1]rnla2[l−1]⋯rnlanl−1[l−1]]\frac{\partial E}{\partial w^{[l]}} = \begin{bmatrix} r_1 a^{[l-1]}_1 & r_1 a^{[l-1]}_2 & \cdots & r_1 a^{[l-1]}_{n^{l-1}} \\ r_2 a^{[l-1]}_1 & r_2 a^{[l-1]}_2 & \cdots & r_2 a^{[l-1]}_{n^{l-1}} \\ \vdots & \vdots & \ddots & \vdots \\ r_{n^l}a^{[l-1]}_1 & r_{n^l}a^{[l-1]}_2 & \cdots & r_{n^l}a^{[l-1]}_{n^{l-1}} \end{bmatrix} ∂w[l]∂E​=​r1​a1[l−1]​r2​a1[l−1]​⋮rnl​a1[l−1]​​r1​a2[l−1]​r2​a2[l−1]​⋮rnl​a2[l−1]​​⋯⋯⋱⋯​r1​anl−1[l−1]​r2​anl−1[l−1]​⋮rnl​anl−1[l−1]​​​

This matrix is equal to:

[r1r2⋮rnl]⋅[a1[l−1]a2[l−1]⋯anl−1[l−1]]\begin{bmatrix} r_1\\r_2\\\vdots\\r_{n^l} \end{bmatrix} \cdot \begin{bmatrix} a^{[l-1]}_1 & a^{[l-1]}_2 & \cdots & a^{[l-1]}_{n^{l-1}} \end{bmatrix} ​r1​r2​⋮rnl​​​⋅[a1[l−1]​​a2[l−1]​​⋯​anl−1[l−1]​​]

We can therefore use a matrix-operation library such as NumPy to accelerate the calculation.

Partial Derivative of the Error with Respect to a Bias

This is relatively simple because ∂zj[l]∂bj[l]=1\frac{\partial z^{[l]}_j}{\partial b^{[l]}_j}=1∂bj[l]​∂zj[l]​​=1, as shown earlier:

∂E∂bj[l]=∂zj[l]∂bj[l]⋅∂aj[l]∂zj[l]⋅∂E∂aj[l]=1⋅σ′(zj[l])⋅∂E∂aj[l]=rj\begin{align*} \frac{\partial E}{\partial b^{[l]}_j} &= \frac{\partial z^{[l]}_j}{\partial b^{[l]}_j}\cdot\frac{\partial a^{[l]}_j}{\partial z^{[l]}_j}\cdot\frac{\partial E}{\partial a^{[l]}_j}\\ &=1\cdot\sigma^\prime(z^{[l]}_j)\cdot\frac{\partial E}{\partial a^{[l]}_j}\\ &=r_j \end{align*} ∂bj[l]​∂E​​=∂bj[l]​∂zj[l]​​⋅∂zj[l]​∂aj[l]​​⋅∂aj[l]​∂E​=1⋅σ′(zj[l]​)⋅∂aj[l]​∂E​=rj​​

The error derivative with respect to the bias equals the rjr_jrj​ used above. Implementations therefore generally calculate this first, then substitute rjr_jrj​ into the weight formula.

Partial Derivative of the Error with Respect to the Previous Layer’s Input

Look again at the multi-node network, now focusing on the effect of a single node in layer l−1l-1l−1 on the later nodes:

    graph LR
     l1["(l-1)1"]  ===> lp1["l1"] & lp2["l2"] & lp3["l3"]

ai[l−1]a^{[l-1]}_iai[l−1]​ can affect every zj[l]z^{[l]}_jzj[l]​. If layer lll is regarded as a function that receives nl−1n^{l-1}nl−1 values ai[l−1]a^{[l-1]}_iai[l−1]​ and outputs nln^lnl values zj[l]z^{[l]}_jzj[l]​, then every input variable is changing. The result is not a partial differential but a total derivative[1].

By the definition of a total derivative, add the partial derivative with respect to every parameter. In this example:

∂E∂ai[l]=∑j=1nl+1(∂zj[l+1]∂ai[l]⋅∂aj[l+1]∂zj[l+1]⋅∂E∂aj[l+1])\frac{\partial E}{\partial a^{[l]}_i}=\sum_{j=1}^{n^{l+1}}\left(\frac{\partial z^{[l+1]}_j}{\partial a^{[l]}_i}\cdot\frac{\partial a^{[l+1]}_j}{\partial z^{[l+1]}_j}\cdot\frac{\partial E}{\partial a^{[l+1]}_j}\right) ∂ai[l]​∂E​=j=1∑nl+1​(∂ai[l]​∂zj[l+1]​​⋅∂zj[l+1]​∂aj[l+1]​​⋅∂aj[l+1]​∂E​)

The factor ∂zj[l+1]∂ai[l]\frac{\partial z^{[l+1]}_j}{\partial a^{[l]}_i}∂ai[l]​∂zj[l+1]​​ needs careful handling. Remember that wji[l+1]w^{[l+1]}_{ji}wji[l+1]​ is the edge connecting node jjj in layer l+1l+1l+1 and node iii in layer lll.

Since

zj[l+1]=wji[l+1]ai[l]+bj[l+1],z^{[l+1]}_j=w^{[l+1]}_{ji}a^{[l]}_i+b^{[l+1]}_j, zj[l+1]​=wji[l+1]​ai[l]​+bj[l+1]​,

we obtain

∂zj[l+1]∂ai[l]=wji[l+1].\frac{\partial z^{[l+1]}_j}{\partial a^{[l]}_i}=w^{[l+1]}_{ji}. ∂ai[l]​∂zj[l+1]​​=wji[l+1]​.

The factor ∂aj[l+1]∂zj[l+1]∂E∂aj[l+1]\frac{\partial a^{[l+1]}_j}{\partial z^{[l+1]}_j}\frac{\partial E}{\partial a^{[l+1]}_j}∂zj[l+1]​∂aj[l+1]​​∂aj[l+1]​∂E​ was explained earlier: it equals rjr_jrj​, the error derivative with respect to the bias.

Rewriting the complete expression gives:

∂E∂ai[l]=∑j=1nl+1(rjwji[l+1]).\frac{\partial E}{\partial a^{[l]}_i}=\sum_{j=1}^{n^{l+1}}\left(r_jw^{[l+1]}_{ji}\right). ∂ai[l]​∂E​=j=1∑nl+1​(rj​wji[l+1]​).

Now consider how to obtain ∂E∂aj[l]\frac{\partial E}{\partial a^{[l]}_j}∂aj[l]​∂E​ with matrix operations. One workable method multiplies rjr_jrj​ and wji[l+1]w^{[l+1]}_{ji}wji[l+1]​.

We accumulate over subscript jjj. If w[l+1]w^{[l+1]}w[l+1] is placed on the left, its jjj coordinate must increase with the column number; in matrix multiplication A×BA\times BA×B, rows of AAA are dotted with columns of BBB. If rrr is placed on the right, its jjj subscript must increase with the row number.

Because jjj in w[l+1]w^{[l+1]}w[l+1] originally increases by row, transpose it. The final result is:

∂E∂a[l]=(w[l+1])T×r,\frac{\partial E}{\partial a^{[l]}}=(w^{[l+1]})^T\times r, ∂a[l]∂E​=(w[l+1])T×r,

where rrr is a column vector.

Implementation

This section uses the backpropagation algorithm derived above to implement a simple fully connected neural network, then uses that network to recognize handwritten digits in the MNIST dataset.

Data Preprocessing

To be honest, the MNIST dataset is rather troublesome. It uses a binary storage format, so reading its contents takes some work.

The struct package may look confusing. It is simply a class specialized for processing binary data.

This statement reads two four-byte unsigned integers in big-endian byte order from lfile. In >II, > indicates that the file uses big-endian byte order, while I indicates a four-byte unsigned integer.

The following np.fromfile serves a similar purpose, directly converting the binary file into a NumPy array. No byte order is specified, perhaps because NumPy uses big-endian by default.

Individual pixels in MNIST are integers in [0,255][0,255][0,255]. We want floating-point values in [0,1][0,1][0,1], so divide by 255 before returning.

Keeping images in [0,1][0,1][0,1] is important because passing a relatively large number to $sigmoid` can cause overflow. Although each layer’s initial weights are randomly generated between -1 and 1, a layer can still sometimes output a large value. Sigmoid is defined as:

σ(x)=11+e−x.\sigma(x)=\frac{1}{1+e^{-x}}. σ(x)=1+e−x1​.

The final preprocessing step converts labels into one-hot form, making it convenient to calculate the gradient of the error over the complete network. np.eye(x) generates an x×xx\times xx×x identity matrix, so $np.eye(10)[labels[i]]is naturally the one-hot encoding corresponding tolabels[i]`.

The layer Class

A single neural-network layer is fundamentally a function that receives a vector and outputs a vector. The function depends on many variables, including weights and biases, so we want a class that stores them.

Backpropagation also needs the variables stored in the class. It is best to implement a function that receives the derivative of the error with respect to the current layer’s output, along with other necessary data, and returns the derivative of the error with respect to the previous layer’s output.

Finally, the layer needs an interface for updating its weights and biases. If other layer types use data other than weights and biases, an abstract class can represent the data of different layers.
