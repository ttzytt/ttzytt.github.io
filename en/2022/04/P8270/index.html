1: Problem Statement

You are given two strings, sss and ttt (the lengths of both sss and ttt do not exceed 10510^5105). You are also given some queries (the number of queries does not exceed 10510^5105). Each query is a subset of the lowercase letters 'a' through 'r'. For every query, determine whether sss and ttt are equal when they contain only the letters given in the query.

2: Analysis

2.1: Brute Force

It is easy to think of a brute-force method. For each query, consider only the characters contained in the set and compare the two strings. However, this requires traversing the strings again for every query, resulting in n2n^2n2 complexity (nnn is both the number of queries and the string length). This method can obtain partial points.

But how can we get the remaining points?

2.2: Simplifying the Problem

Directly solving this problem may be too complicated. We can try simplifying it first and then generalizing the simplified solution to the original problem.

First consider the case where a query contains only two letters. Let them be aaa and bbb. How do we determine whether two strings containing only aaa and bbb are equal?

The first thing to consider is whether the numbers of aaas and bbbs in the two strings are equal. If either count is different, the two strings must be different.

Next, we need to consider the positions of every aaa and bbb in the strings. If both the positions and counts are correct, the two strings must be equal.

When comparing the positions of aaa and bbb, we certainly cannot directly compare their indices, because we are comparing their positions after keeping only aaa and bbb in the two strings. Their indices must change after other characters are removed.

After removing other characters, the index of each character in a string is actually the number of aaas before it plus the number of bbbs before it (all other characters have been removed).

Of course, checking the indices of every aaa and bbb in order takes too much time, so we can optimize. For example, it is enough to check that the positions of one of aaa and bbb are all equal. Since the numbers of aaas and bbbs are equal in both strings, once the positions of one character are determined, the positions of the other are determined as well (every position that is not aaa must be bbb).

This check can be simplified further. We can consider only the number of bbbs before each aaa. Consider the string "baa". If we use the number of bbbs before an aaa as the index of aaa, the indices of the two aaas are the same. If we exchange these two aaas, the string remains the same, so the fact that their indices are equal does not affect our determination of the positions of aaa.

In summary, two strings containing only two characters (assume they are aaa and bbb) are equal only if:

  • The numbers of aaas and bbbs are equal.
  • The number of bbbs before every aaa is equal.

Why do we use this method to determine whether strings are equal?

Because prefix sums let us quickly determine whether two strings containing only two characters are equal.

  • The number of each character before every position in the original string.
  • Every position of each character in the original string.

2.3: Considering the Original Problem

Now that we can quickly determine whether two strings containing only two characters are equal, let us consider how to apply this to the original problem.

Suppose the two strings were originally equal. There are several ways to make them different:

  • Add a character.
  • Delete a character.
  • Exchange two characters (exchanging two equal characters is equivalent to not exchanging them).

Note: for convenience, call the function that determines whether two strings containing only characters aaa and bbb are equal isok(a, b).

For the first two changes, the count of some character in the two strings must change. Suppose the added or deleted character is aaa. Then isok(a, other character) must return false, because the counts of aaa in sss and ttt are no longer equal.

Now consider exchanging characters. Suppose the exchanged characters are aaa and bbb. Then isok(a, b) must also return false, because the strings consisting only of aaa and bbb in sss and ttt must be different.

Therefore, for each query, we only need to enumerate every pair of different characters included in the query and determine whether sss and ttt are equal when containing only those two characters.

Remember to store every isok(a, b) result so that it does not need to be recalculated later.

2.3: Complexity Analysis

  • Preprocessing: O(n)O(n)O(n).
  • isok(a, b): because we need to know the number of bbbs before every aaa, we enumerate aaa, so the complexity is the number of aaas.
  • Processing all isok(a, b) results: O(n)O(n)O(n) (nnn is the string length), because we enumerate every aaa and bbb, and the sum of all their counts is the string length.
  • Queries: (number of letters in one query)2×number of queries\text{(number of letters in one query)}^2 \times \text{number of queries}(number of letters in one query)2×number of queries. Since every isok(a, b) result has already been processed, enumerating two different characters in a query only requires returning the result in O(1)O(1)O(1) time. We enumerate (number of letters in one query)2\text{(number of letters in one query)}^2(number of letters in one query)2 pairs in total.
