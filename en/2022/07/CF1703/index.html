F. Yet Another Problem About Pairs Satisfying an Inequality

思路（来源于官方题解）

观察题目中的不等式 ai<i<aj<ja_i < i < a_j < jai​<i<aj​<j。可以发现，对于数组中的任意元素，只要不符合 ax<xa_x < xax​<x，那就绝对不会和任何的元素组成一个合法的数对。所以我们可以直接跳过不符合 ax<xa_x < xax​<x 的元素。

我们可以把这个不等式拆成三个部分 ai<i,i<aja_i < i, i < a_jai​<i,i<aj​ 和 aj<ja_j < jaj​<j。

那对于所有符合 ax<xa_x < xax​<x 的元素，第一个和最后一个不等式已经满足了，只要找到满足 i<aji < a_ji<aj​ 的元素，就可以构成一个合法数对了。

我们设 aaa 去掉不满足 ax<xa_x < xax​<x 的数组为 bbb （听起来有点奇怪，但是 bbb 中的每个元素的下标是跟 aaa 一样的）。

比如我们说数组中的一个元素有值和下标两个属性，并且用这样的方式标记：(val,id)(val, id)(val,id)，那么如果 aaa 数组是：

a=(1,1) (1,2) (2,3) (3,4) (8,5) (2,6) (1,7) (4,8)a = (1, 1) \ (1, 2)\ (2, 3)\ (3, 4)\ (8, 5)\ (2, 6)\ (1, 7)\ (4, 8)\\ a=(1,1) (1,2) (2,3) (3,4) (8,5) (2,6) (1,7) (4,8)

那去掉所有 val>=idval >= idval>=id 的，就能得到 bbb:

b=(1,2) (2,3) (3,4) (2,6) (1,7) (4,8)b = (1, 2)\ (2, 3)\ (3, 4)\ (2, 6)\ (1, 7)\ (4, 8)\\ b=(1,2) (2,3) (3,4) (2,6) (1,7) (4,8)

那只要我们对于每个 bjb_jbj​ 找到所有符合的 iii，就可以解决本题。

其中不难发现 iii 是单调递增的，所以可以使用二分来找最大的，小于 bjb_jbj​ 的 iii。那么 bbb 中所有 ididid 小于 iii 的元素（以及 iii 自身）都可以跟 aja_jaj​ 构成一个合法数对。

除了二分法，我们还可以用树状数组来找到 bbb 中所有 ididid 小于一个特定值的元素的数量。

具体来说，我们可以用树状数组维护一个前缀和数组，然后遍历 bbb 中的元素，每次都做 upd(id) 的操作。也就是使查询所有大于等于 ididid 的数时，查到的值都增加 111。

这样，在树状数组中查询某个 bj−1b_j - 1bj​−1 的时候，就会返回比 bjb_jbj​ 小的所有 iii 了。

当然，用差分的方法，也可以得到和树状数组相同的前缀和数组。而且本题不需要我们在得到这个前缀和数组后做别的更新，所以差分可以用 O⁡(n)\operatorname{O}(n)O(n) 的复杂度解决本题。

G. Good Key, Bad Key

思路（来源于官方题解）

我们能发现，对于所有的盒子，交错的使用好钥匙和坏钥匙总是更加不合算的。

并且，连续的，在前面的盒子使用好钥匙会更合算。（或者说使用好钥匙作为前缀）。

假设我们在一个好钥匙之前使用了一个坏钥匙，那么我们获得的收益是：

⌊ai2⌋+(⌊ai+12⌋−k)\lfloor \frac{a_i}{2} \rfloor + (\lfloor \frac{a_{i + 1}}{2} \rfloor - k) ⌊2ai​​⌋+(⌊2ai+1​​⌋−k)

但如果先使用好钥匙，后使用坏钥匙，获得的收益是：

(ai−k)+⌊ai+12⌋(a_i - k) + \lfloor \frac{a_{i + 1}}{2} \rfloor (ai​−k)+⌊2ai+1​​⌋

很明显，先使用好钥匙更合算。

更直观一点的解释是，不管先使用哪种钥匙，都会减去 kkk 的收益，但是如果先使用坏钥匙，我们会把两个盒子的收益减半，但如果先使用好钥匙，就只会把一个盒子的收益减半。

所以我们只会在最后的部分使用坏钥匙，在某些 kkk 比较大的情况下，可能相比减去 kkk，把 aia_iai​ 减半更合算。

所以只需要枚举一个使用好钥匙和坏钥匙的分割点，在这个点前面全部使用好钥匙，后面全部使用坏钥匙。

我们设这个分割点为 xxx。

那么在分割点 xxx 后面使用坏钥匙，每个盒子的收益就会变成：

ax+i=⌊ax+i÷2i⌋a_{x + i} = \lfloor a_{x + i} \div 2^{i} \rfloor ax+i​=⌊ax+i​÷2i⌋

能发现这个 2i2^i2i 会增长的很快，在某个点之后，aia_iai​ 就会变成 000。那么在这个点之后我们就没必要再计算了。

因为最大的 aia_iai​ 为 10910^9109，所以 log2(109)≈30log_2(10^9) \approx 30log2​(109)≈30 之后就没必要计算了。（或者你可以想，一直右移一个数，那么过了某个点整个数的二进制形式就没有 111 了）。
