C. Mark and His Unfinished Essay

Approach

Given these constraints, we obviously cannot actually copy the string, so we need to find another method.

We can see that every segment newly appended to the end of the string has an identical counterpart earlier in the string, located by an offset.

For example, consider the final insertion in the first sample:

mark mark mar rkmark\texttt{mark\ mark\ mar\ } \color{red}{\texttt{rkmark}} mark mark mar rkmark

If every letter in this rkmark\texttt{rkmark}rkmark is moved 9 positions backward, we find another rkmark\texttt{rkmark}rkmark, as follows:

ma rkmark mar rkmark\texttt{ma\ } \textcolor{red}{\texttt{rkmark\ }}\texttt{mar\ } \textcolor{red}{\texttt{rkmark}} ma rkmark mar rkmark

Therefore, we can maintain a triple (l,r,d)(l, r, d)(l,r,d) indicating that the characters in the interval [l,r][l, r][l,r] are completely identical to those in the interval [l−d,r−d][l - d, r - d][l−d,r−d].

Then, whenever querying position xxx, we can continually subtract the corresponding ddd until xxx lies within the range of the initial string.

D. Mark and Lightbulbs

Approach

First, let us simulate the fourth sample:

000101↓001101↓011101↓011001↓010001↓010011\texttt{000101} \\ \downarrow\\ \texttt{00}\textcolor{red}{\texttt{1}}\texttt{101}\\ \downarrow\\ \texttt{0}\textcolor{red}{\texttt{1}}\texttt{1101}\\ \downarrow\\ \texttt{011}\textcolor{red}{\texttt{0}}\texttt{01}\\ \downarrow\\ \texttt{01}\textcolor{red}{\texttt{0}}\texttt{001}\\ \downarrow\\ \texttt{0100}\textcolor{red}{\texttt{1}}\texttt{1}\\ 000101↓001101↓011101↓011001↓010001↓010011

Note: The positions marked in red indicate changes.

We can see that during this process, we can only lengthen or shorten a segment consisting of 111s, such as 1\texttt{1}1 or 111\texttt{111}111 (of course, viewed the other way around, it can be described as a segment consisting of 000s), rather than creating a new “111 segment” out of thin air. This is because only when a 111 changes to a 000, or a 000 changes to a 111, will si−1s_{i - 1}si−1​ and si+1s_{i + 1}si+1​ differ, allowing us to change sis_isi​.

Therefore, if strings sss and ttt have different numbers of segments, transforming sss into ttt must be impossible.

We can see that in each operation, we can move the beginning or end of a “111 segment” by one position. We can therefore use this fact to calculate the number of steps required to transform sss into ttt.

That is, for every segment in sss and ttt, we calculate the positions at which the segment begins and ends, then calculate the differences between corresponding segment endpoints in sss and ttt. The sum of these differences is the answer.

How do we determine the beginning and end of a segment? They are simply where 000 changes to 111 and where 111 changes to 000. Therefore, we create two arrays, aaa and bbb. After reading sss and ttt, we traverse the two strings. Whenever si≠si+1s_i \ne s_{i + 1}si​=si+1​, we put iii into aaa (and do the same for ttt and bbb). In this way, aaa and bbb store all segment endpoints in the two strings.
