C. Train and Queries

Problem statement

Problem links: (Codeforces, Luogu).

You are given an array uuu of length n(1≤n≤2⋅105)n (1\le n \le 2 \cdot 10 ^ 5)n(1≤n≤2⋅105) representing all train stations. A train can travel only from a station on the left to a station on the right. In other words, it starts at u1u_1u1​, then proceeds to u2, u3u_2,\ u_3u2​, u3​, and finally reaches unu_nun​.

You are then given k(1≤k≤2⋅105)k (1\le k \le 2 \cdot 10 ^ 5)k(1≤k≤2⋅105) queries. Each query contains two integers aia_iai​ and bib_ibi​ and asks whether it is possible to board at station aia_iai​ and travel by train to station bib_ibi​.

For example, suppose the array uuu is [3,7,1,5,1,4][3,7,1,5,1,4][3,7,1,5,1,4] and there are the following three queries:

  • a1=3,b1=5a_1 = 3, b_1 = 5a1​=3,b1​=5.
    It is possible to travel from station 333 to station 555 along the route [3,7,1,5][3,7,1,5][3,7,1,5].
  • a2=1,b2=7a_2 = 1, b_2 = 7a2​=1,b2​=7.
    There is no route that travels from station 111 to station 777.
  • a3=3,b3=10a_3 = 3, b_3 = 10a3​=3,b3​=10.
    There is no route from station 333 to station 101010, because station 101010 does not exist at all.

D. Not a Cheap String

Problem statement

Let sss be a string consisting of lowercase Latin letters. Its price is defined as the sum of the positions of all its letters in the alphabet.

For example, the price of the string abca\texttt{abca}abca is 1+2+3+1=71 + 2 + 3 + 1 = 71+2+3+1=7.

You are given a string w(∣w∣≤2⋅105)w (|w| \le 2\cdot 10^5)w(∣w∣≤2⋅105) and an integer ppp. Remove as few letters as possible from the string so that the price of www is at most p(1≤p≤5 200 000)p (1 \le p \le 5\ 200\ 000)p(1≤p≤5 200 000). Note that you may remove no letters, or you may remove every letter in the string.

Approach

This problem is actually about as difficult as the previous one. Since the problem asks us to delete as few letters as possible, we directly choose letters that contribute the most to the price and delete them until the total price becomes at most ppp.

For the implementation, we can again use a map to create a mapping from each character to its number of occurrences—in other words, a bucket.

We then traverse this map in reverse order, so the characters visited first make the greatest contribution to the price. During the traversal, if the current price is greater than ppp, we delete the current character. Whenever we delete a character, we also decrement its occurrence count by one.

Finally, to produce the output, we traverse the original string. If the corresponding character still has an occurrence in the bucket, we output it and decrement the remaining count; otherwise, we omit it.

E. Split Into Two Sets

Problem statement

You are given nnn pairs, where nnn is even and 2≤n≤2⋅1052 \le n \le 2 \cdot 10^52≤n≤2⋅105. Every number in every pair is between 111 and nnn.

Determine whether these pairs can be divided into two sets such that no number is repeated within either set.

For example, consider the four pairs {1,4},{1,3},{3,2},{4,2}\{1, 4\}, \{1, 3\}, \{3, 2\}, \{4, 2\}{1,4},{1,3},{3,2},{4,2}.

They can be assigned as follows:

  • The first set contains the pairs {1,4}\{1, 4\}{1,4} and {3,2}\{3, 2\}{3,2}, while the second set contains {1,3}\{1, 3\}{1,3} and {4,2}\{4, 2\}{4,2}.

Approach

At first glance, this looks like a greedy problem: put a pair into the first set whenever possible; if that is impossible, put it into the other set; if neither placement works, output NO\texttt{NO}NO. However, this is an E problem, so it is not that simple. (Do not copy me by submitting a greedy solution immediately and then spending ages wondering why it is wrong.)

To prove that this greedy method is wrong, we only need a counterexample. As an aside, the samples for this problem are rather misleading because the greedy method passes all of them.

Consider the following input:

Suppose the first set is AAA and the second is BBB. Under the greedy approach, the first two pairs, {1,2}\{1,2\}{1,2} and {5,4}\{5,4\}{5,4}, can be placed into AAA. At the third pair, the 222 in {2,3}\{2,3\}{2,3} conflicts with the 222 in {1,2}\{1,2\}{1,2}, so we put that pair into BBB.

For the fourth pair {4,3}\{4,3\}{4,3}, however, we find that it conflicts no matter which set receives it.

Nevertheless, this input can legally be divided into two sets:

A:{1,2} {4,3} {5,6}B:{2,3} {5,4} {6,1}A: \{1, 2\} \ \{4, 3\} \ \{5, 6\}\\ B: \{2, 3\} \ \{5, 4\} \ \{6, 1\} A:{1,2} {4,3} {5,6}B:{2,3} {5,4} {6,1}

We can break each pair apart and consider its individual numbers.

Start with 111. Two of the pairs contain 111: {1,2}\{1,2\}{1,2} and {6,1}\{6,1\}{6,1}. Because both pairs contain 111, they certainly cannot belong to the same set.

Apply the same reasoning to 222. The two pairs containing 222 are {2,3}\{2,3\}{2,3} and {1,2}\{1,2\}{1,2}, so they must also belong to different sets.

Listing the pairs that contain each number from 111 through nnn in this way gives:

1→{1,2} {6,1}2→{2,3} {1,2}3→{2,3} {4,3}4→{4,3} {5,4}5→{5,4} {5,6}6→{5,6} {6,1}1 \to \{1, 2\} \ \{6, 1\}\\ 2 \to \{2, 3\} \ \{1, 2\}\\ 3 \to \{2, 3\} \ \{4, 3\}\\ 4 \to \{4, 3\} \ \{5, 4\}\\ 5 \to \{5, 4\} \ \{5, 6\}\\ 6 \to \{5, 6\} \ \{6, 1\} 1→{1,2} {6,1}2→{2,3} {1,2}3→{2,3} {4,3}4→{4,3} {5,4}5→{5,4} {5,6}6→{5,6} {6,1}

When we check these conditions, no contradiction appears, and the assignment shown earlier can be derived from them.

Viewed this way, the problem tells us that two objects must be in different sets and asks whether all such rules can be satisfied. Is that not precisely a disjoint-set union structure with logical relationships?

If you are not familiar with this technique, you can look at these problems:

  • P2024 [NOI2001] Food Chain

Indeed, this problem can be solved with a disjoint-set union structure that maintains logical relationships; this is how tourist solved it.

However, we can also approach it from the perspective of graph theory.

If we connect the two numbers in every pair with an edge, we obtain a graph like this:

For the same reason as before, consider a number such as 222. The two pairs containing 222, namely {2,3}\{2,3\}{2,3} and {1,2}\{1,2\}{1,2}, cannot be placed into the same set.

In terms of edges, vertex 222 is incident to two edges, and we cannot select both of those edges for the same set.

The only way to meet this requirement is therefore to assign the edges alternately to the two sets.

For example:

Here, edges drawn as <--> and edges drawn as <==> represent the two different sets to which the pairs of endpoint vertices will be assigned.

We can now consider separately whether different graph shapes meet the requirement.

First, if a vertex is incident to three or more edges, alternating between the two sets is impossible.

For example:

To place the three edges to B,C,DB,C,DB,C,D into two sets, at least one of the pairs ABABAB and ACACAC, ACACAC and ADADAD, or ABABAB and ADADAD must belong to the same set. This violates the alternating requirement because AAA inevitably appears twice in that set.

Second, if a connected component is only a path, alternating its edges between the two sets always satisfies the requirement.

Finally, if a component is a cycle with an even number of edges, as in the earlier example, its edges can certainly alternate. A cycle with an odd number of edges cannot satisfy the condition.

The parity of a cycle can be checked rather directly. Give each edge a color chosen from two colors, and use DFS to traverse the cycle.

During traversal, try to color consecutive edges alternately. If this alternating coloring fails, the cycle must be odd, and the converse is also true. If the edges can be colored alternately, the two colors must occur equally often, so the cycle must be even.

There is one more implementation detail to note. The graph we construct is not necessarily connected, so we must attempt a DFS from every vertex. In addition, constructing the graph directly from the input can produce parallel edges, which we need to avoid.

F. Equate Multisets

Preface: the solution in this explanation refers to this video.

Problem statement

A multiset is a special kind of set whose elements may repeat. Like an ordinary set, the order of its elements does not matter. Two multisets are equal when every element occurs the same number of times in both.

For example, {2,2,4}\{2,2,4\}{2,2,4} and {2,4,2}\{2,4,2\}{2,4,2} are equal, whereas {1,2,2}\{1,2,2\}{1,2,2} and {1,1,2}\{1,1,2\}{1,1,2} are not.

You are given two multisets aaa and bbb, each containing n(1≤n≤2⋅105)n (1 \le n \le 2\cdot10^5)n(1≤n≤2⋅105) integers.

In one operation, you may double one element of bbb or halve it with rounding down. In other words, for an element xxx of bbb, you may perform either of the following operations:

  • Replace xxx with 2x2x2x.
  • Replace xxx with ⌊x2⌋\lfloor \frac{x}{2} \rfloor⌊2x​⌋.

Note that no operation may be performed on multiset aaa.

Determine whether multiset bbb can be made equal to aaa after any number of operations, including zero operations.

Some properties

The operations ×2\times 2×2 and ⌊÷2⌋\lfloor \div 2 \rfloor⌊÷2⌋ correspond to bitwise left and right shifts. For example, the binary representation of 555 is (101)2(101)_2(101)2​, while that of 5×25\times25×2 is (1010)2(1010)_2(1010)2​. Compared with 555, the binary form of 101010 has an additional 000 at the end. Conversely, 10÷210\div210÷2 is 555, whose binary representation has one fewer trailing 000 than that of 101010.

Thus, shifting left is equivalent to multiplying by two, and shifting right is equivalent to floor division by two.

From this we can observe a property: trailing zeroes of elements in multisets aaa and bbb are unimportant. Strictly speaking these are multisets, but I will call them sets here for convenience.

I will explain both what a trailing zero is and what “unimportant” means.

Consider the number 404040, whose binary representation is (101000)2(101000)_2(101000)2​. The binary form of 404040 has three zeroes at its end. These three zeroes are the trailing zeroes of 404040.

By “unimportant,” I mean the following.

Let α∈a\alpha \in aα∈a and β∈b\beta \in bβ∈b, and let α′\alpha^\primeα′ and β′\beta^\primeβ′ be the numbers obtained from α\alphaα and β\betaβ, respectively, after removing their trailing zeroes. If the two allowed operations can transform β\betaβ into α\alphaα, then they can also transform β′\beta^\primeβ′ into α′\alpha^\primeα′.

This is because left and right shifts can append any number of zeroes to the end of β′\beta^\primeβ′ or remove any number of them.

We can therefore turn β′\beta^\primeβ′ back into β\betaβ. We already know that β\betaβ can be transformed into α\alphaα. After that, removing some zeroes from the current number yields α′\alpha^\primeα′.

For convenience in the subsequent computation, we can therefore strip all trailing zeroes from the input elements immediately.

There is another property:

We can transform β′\beta^\primeβ′ into α′\alpha^\primeα′ if and only if the binary representation of α′\alpha^\primeα′ is a prefix of the binary representation of β′\beta^\primeβ′.

First, let us clarify what a prefix means for a binary representation. Consider the numbers 999 and 757575, whose binary forms are (1001)2(1001)_2(1001)2​ and (1001011)2(1001011)_2(1001011)2​, respectively.

From the perspective of strings, 1001\texttt{1001}1001 is a prefix of 1001011\texttt{1001011}1001011. The reason β′\beta^\primeβ′ can be transformed into α′\alpha^\primeα′ is the right-shift operation: we may remove bits from the end of β′\beta^\primeβ′ until it becomes any prefix of its own binary representation.

It is also evident that if α′>β′\alpha^\prime > \beta^\primeα′>β′, then α′\alpha^\primeα′ cannot be a prefix of the binary representation of β′\beta^\primeβ′. Consequently, β′\beta^\primeβ′ cannot be transformed into α′\alpha^\primeα′.

Implementation

With these properties in hand, we can devise a somewhat unusual method.

First, store the elements of multiset aaa in an array and the elements of multiset bbb in a priority queue. Before storing an element, strip its trailing zeroes.

Then sort aaa in ascending order. After that, we can perform the following operations:

As the code shows, on each iteration of this while loop, lalala and lblblb are the largest elements currently present in aaa and bbb, respectively.

There are three cases:

  1. la>lbla > lbla>lb: In this case, we can immediately output NO, because lalala is certainly not a prefix of the binary representation of lblblb, as explained earlier. Moreover, lblblb is already the largest element in all of bbb. If lblblb cannot be transformed into lalala, no other element of bbb can possibly be transformed into lalala either.
  2. la=lbla = lbla=lb: Since the two elements are equal, both can be removed from their multisets. When the multisets become empty, we can output YES. This is why the code contains a.pop_back();.
  3. la<lbla < lbla<lb: At this point, we do not know whether lalala is a prefix of lblblb, but it might be. We therefore right-shift lblblb by one bit, turning it into its longest proper prefix, and later check whether ⌊lb2⌋\lfloor \frac{lb}{2} \rfloor⌊2lb​⌋ matches another element of aaa.

For the third case, could right-shifting lblblb immediately and placing it back into the priority queue destroy a match in which the original lblblb could have matched some other element of aaa?

No. The largest element of aaa is already smaller than lblblb, so every other element of aaa is also smaller. Therefore, no other element can be equal to the original value of lblblb.
