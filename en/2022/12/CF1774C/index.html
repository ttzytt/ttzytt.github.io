To complain a little, the official solution was rather difficult to understand. I remained quite confused after reading it for a long time (actually, I am also just not good enough). After understanding it, I felt that this problem was quite clever, so I came to write a solution.

Approach

We first need to make an observation: for the string sss, its final consecutive segment does not increase the number of possible winners. For example, when s=0011s = \texttt{0011}s=0011, the ending 11\texttt{11}11 does not increase the number of possible winners.

Why? Suppose that, after any number of matches, the set of possible player combinations is ttt. Then, for any x∈tx \in tx∈t, after any number of consecutive matches in environment 111, the final winner must be the player in xxx with the highest temperature. This is because every remaining player must play continuously in environment 111, and the only player who can win every match must be the largest. Similarly, after the players in ttt play any number of consecutive matches in environment 000, the final winner must be the player in xxx with the lowest temperature.

For example, when s=111s = \texttt{111}s=111, player 4 must be the final winner.

Thus, if the ending segment is 111 (the same applies to an ending of 000; for convenience, the examples below use 111), we only need to calculate how many player combinations with different maximum values (player temperatures) can be constructed by the preceding part. This tells us the answer for sss at its current length.

Now consider how to construct the maximum number of player combinations with different maximum values. If there are nnn players, then before any matches, the maximum value is nnn. To make the maximum value different, we can only remove the current maximum.

Special Case

The preceding description may be relatively abstract. The example s=0011s = \texttt{0011}s=0011 makes it easier to understand.

For the first 000, every player other than the one with the lowest temperature in the player combination can be removed (that lowest-temperature player can win no matter what). We can make the player with the lowest temperature play against any other player, giving the following cases:

1:123452:123453:123454:123451 : \texttt{1234} \cancel{\texttt{5}} \\ 2 : \texttt{123} \cancel{\texttt{4}} \texttt{5} \\ 3 : \texttt{12} \cancel{\texttt{3}} \texttt{45} \\ 4 : \texttt{1} \cancel{\texttt{2}} \texttt{345} \\ 1:123452:123453:123454:12345

Observation shows that only the first case changes the maximum value (why? Because it removes the maximum). In the other cases, a consecutive segment of numbers at the end must be removed before the maximum value can change.

This is where the second 000 has an effect. In the second case, it can remove 555, making the maximum value of the player combination become 333. Following this pattern, we can generalize the following conclusion: suppose the length of the consecutive segment before the ending segment is lll. The number of player combinations it can produce with different maximum values is l+1l + 1l+1. Specifically, the possible range of maximum values is [n−l,n][n - l, n][n−l,n]. (The +1+1+1 is because we can choose not to change the original maximum value.)

At this point, we can already find the answer when sss contains only two consecutive segments. It is n−kn - kn−k, where k=∣s∣−lk = |s| - lk=∣s∣−l represents the length of the ending segment.

Generalization

Now let us change the example to s=1011s = \texttt{1011}s=1011 and see whether the conclusion still holds (sss contains more than one segment). Similarly, we can list the possible player combinations after the first match. Because the first environment is 111, only players other than the maximum can be removed:

1:123452:123453:123451:123451 : \texttt{123} \cancel{\texttt{4}} \texttt{5} \\ 2 : \texttt{12} \cancel{\texttt{3}} \texttt{45} \\ 3 : \texttt{1} \cancel{\texttt{2}} \texttt{345} \\ 1 : \cancel{\texttt{1}} \texttt{2345} \\ 1:123452:123453:123451:12345

Although none of these cases changes the maximum value of the player combination, we only need to play one more match in the following 000 environment and remove 555 to produce two new maximum values. In the first case, the maximum becomes 333; in the second case, it becomes 222. There are also n−kn - kn−k possible answers in total.

What if there are even more segments? For example, s=01011s = \texttt{01011}s=01011. The conclusion still holds. We can regard 010\texttt{010}010 as a group of environments, in which 000 can remove any player in the range [2,n][2, n][2,n], while 111 can remove any player in [1,n−1][1, n - 1][1,n−1]. Combining these two kinds of environments allows us to choose any three players from [1,n][1, n][1,n] to remove, constructing four different maximum values (depending on how many of the players with the highest temperatures are removed from the beginning).
